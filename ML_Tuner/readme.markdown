# ML 参数调优模块完整设计方案
## 一、模型定位与职责边界

```text
┌─────────────────────────────────────────────────────────────────────┐
│                     系统全局调优分工                                   │
│                                                                     │
│  ┌──────────────────────┐        ┌──────────────────────┐          │
│  │     DS2 Controller    │        │     ML Tuner          │          │
│  │     (快循环)           │        │     (慢循环)           │          │
│  │                       │        │                       │          │
│  │  What: 算子并行度      │        │  What: 非并行度参数    │          │
│  │  When: 每15秒决策一次  │        │  When: 每10分钟决策    │          │
│  │  How:  解析式DS2模型   │        │  How:  GP+贝叶斯优化  │          │
│  │  Risk: 低(热调整)      │        │  Risk: 中(需重启)      │          │
│  │  State: 无状态计算     │        │  State: 历史观测积累   │          │
│  └───────────┬──────────┘        └───────────┬──────────┘          │
│              │                                │                     │
│              │  互斥协调: 文件锁 + 状态机       │                     │
│              └────────────┬───────────────────┘                     │
│                           │                                         │
│              ┌────────────▼───────────────────┐                     │
│              │    共享执行通道                   │                     │
│              │    Savepoint → Cancel → Resubmit│                     │
│              └────────────────────────────────┘                     │
└─────────────────────────────────────────────────────────────────────┘

```
### 核心原则
  1. DS2 管并行度, ML Tuner 管其余参数, 参数空间完全正交
  2. 任一方触发重配置时, 另一方必须暂停, 通过文件锁实现互斥
  3. ML Tuner 运行周期远长于 DS2, 是「战略级」调优; DS2 是「战术级」响应

## 二、调优参数空间定义
### 2.1 参数清单与边界

```text
┌───────────────────────────────────────────────────────────────────┐
│                      3 维参数空间 θ                                │
│                                                                   │
│  参数名                │ 类型  │ 范围           │ 步长   │ 默认值  │
│  ─────────────────────┼──────┼───────────────┼───────┼───────  │
│  checkpoint_interval   │ int  │ [5000, 120000]│ 5000  │ 30000ms │
│  buffer_timeout        │ int  │ [1, 200]      │ 10    │ 100ms   │
│  watermark_delay       │ int  │ [1000, 30000] │ 1000  │ 5000ms  │
│                                                                   │
│  搜索空间: 24 × 20 × 30 = 14,400 个离散候选配置                    │
│  → 仍需贝叶斯优化, 因为 3 参数间存在强耦合                          │
│  → 同时可做网格搜索作为对比实验基线                                  │
└───────────────────────────────────────────────────────────────────┘
```

### 2.2 参数耦合关系
```text
┌──────────────────────────────────────────────────────────────┐
│              3 维参数间的耦合关系                               │
│                                                              │
│                 checkpoint_interval                          │
│                 /                  \                          │
│         CP期间增加延迟          CP频率影响I/O开销              │
│               /                      \                       │
│              ▼                        ▼                      │
│     整体延迟 (latency)         CPU/IO 负担                   │
│              ▲                        ▲                      │
│               \                      /                       │
│       窗口触发频率影响延迟     buffer积满触发flush              │
│                 \                  /                          │
│                 watermark_delay                              │
│                       ↕                                      │
│                 buffer_timeout                               │
│                       │                                      │
│                 buffer越小网络包越多                           │
│                 → 吞吐可能下降                                │
│                                                              │
│  三者构成一个紧密耦合的三角:                                    │
│  - checkpoint 越频繁 → 需要更大 buffer 缓解 I/O 冲击          │
│  - watermark 越大 → 窗口触发越慢 → 延迟越高                   │
│  - buffer 越小 → 延迟越低 → 但 checkpoint 对齐压力越大        │
└──────────────────────────────────────────────────────────────┘
```

### 2.3 各参数对系统行为的影响
```text
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│  checkpoint_interval:                                            │
│  ┌───────┬────────────────────────────────────────────────────┐  │
│  │ 值小   │ 快照频繁 → 故障恢复快 → 但 I/O 开销大              │  │
│  │ (5s)  │ → checkpoint 期间算子暂停 → 延迟毛刺频繁             │  │
│  ├───────┼────────────────────────────────────────────────────┤  │
│  │ 值大   │ 快照稀疏 → 故障恢复慢 → I/O 开销小                 │  │
│  │ (120s)│ → 延迟毛刺少 → 但丢数据风险增大                     │  │
│  └───────┴────────────────────────────────────────────────────┘  │
│                                                                  │
│  buffer_timeout:                                                 │
│  ┌───────┬────────────────────────────────────────────────────┐  │
│  │ 值小   │ 网络 buffer 快速 flush → 延迟低                    │  │
│  │ (1ms) │ → 但小包增多, 网络开销大, 吞吐可能下降              │  │
│  ├───────┼────────────────────────────────────────────────────┤  │
│  │ 值大   │ buffer 积满再 flush → 吞吐量高                     │  │
│  │(200ms)│ → 但延迟增大                                       │  │
│  └───────┴────────────────────────────────────────────────────┘  │
│                                                                  │
│  watermark_delay:                                                │
│  ┌───────┬────────────────────────────────────────────────────┐  │
│  │ 值小   │ 窗口触发快 → 实时性好                               │  │
│  │ (1s)  │ → 但迟到数据被丢弃, 结果可能不准                     │  │
│  ├───────┼────────────────────────────────────────────────────┤  │
│  │ 值大   │ 容忍更多乱序 → 结果更准                             │  │
│  │ (30s) │ → 但窗口触发延迟大, 端到端延迟显著增加               │  │
│  └───────┴────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘

```

### 3.3 特征预处理
```text
┌────────────────────────────────────────────────────────────┐
│                    特征预处理流水线                           │
│                                                            │
│  Step 1: 多次采样取中位数                                    │
│  ┌──────────────────────────────────────────────────┐      │
│  │  在观测窗口 (5min) 内, 每 30s 采集一次             │      │
│  │  共 10 个样本, 取中位数 → 消除瞬时毛刺              │      │
│  └──────────────────────────────────────────────────┘      │
│                                                            │
│  Step 2: Min-Max 归一化到 [0, 1]                            │
│  ┌──────────────────────────────────────────────────┐      │
│  │  x_norm = (x - x_min) / (x_max - x_min)          │      │
│  │                                                    │      │
│  │  各特征的 min/max (先验知识):                       │      │
│  │  avg_input_rate:    [0, 500000]   rec/s            │      │
│  │  peak_input_rate:   [0, 1000000]  rec/s            │      │
│  │  input_cv:          [0, 2.0]                       │      │
│  │  active_node_count: [0, 2000]                      │      │
│  │  avg_backlog:       [0, 1000000]  records          │      │
│  │  avg_busy_ratio:    [0, 1.0]                       │      │
│  │  max_busy_ratio:    [0, 1.0]                       │      │
│  │  total_parallelism: [1, 50]                        │      │
│  │  heap_usage_ratio:  [0, 1.0]                       │      │
│  │  latency_p99:       [0, 10000]   ms                │      │
│  │  ckp_avg_dur:       [0, 60000]   ms                │      │
│  │  backpressure:      [0, 1.0]                       │      │
│  │  l2_hit_rate:       [0, 1.0]                       │      │
│  │  l2_occupancy:      [0, 1.0]                       │      │
│  │  l3_hit_rate:       [0, 1.0]                       │      │
│  │  l3_occupancy:      [0, 1.0]                       │      │
│  └──────────────────────────────────────────────────┘      │
│                                                            │
│  Step 3: 参数也做归一化                                      │
│  ┌──────────────────────────────────────────────────┐      │
│  │  θ_norm = (θ - θ_min) / (θ_max - θ_min)          │      │
│  │  → GP 各维度尺度一致, RBF kernel 不被量纲误导      │      │
│  └──────────────────────────────────────────────────┘      │
│                                                            │
│  最终 GP 输入: z = [X_norm(16维), θ_norm(3维)] ∈ R^19     │
└────────────────────────────────────────────────────────────┘
```

## 四、优化目标函数
### 4.1 复合目标
```text
┌──────────────────────────────────────────────────────────────┐
│                   目标函数 J(θ|X)                             │
│                                                              │
│  J = w₁ · f_latency                                         │
│    + w₂ · f_throughput                                       │
│    + w₃ · f_checkpoint                                       │
│    + w₄ · f_resource                                         │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  f_latency    = latency_p99 / SLA                      │  │
│  │               = latency_p99 / 5000                      │  │
│  │               > 1.0 表示违反 SLA                         │  │
│  │               目标: 越小越好                              │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  f_throughput = 1 - (actual_throughput / max_throughput) │  │
│  │               max_throughput 取历史观测中的最大值         │  │
│  │               目标: 越小越好 (吞吐量越大越好)             │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  f_checkpoint = checkpoint_avg_duration / SLA           │  │
│  │               = ckp_dur / 5000                          │  │
│  │               反映 checkpoint 对系统性能的拖累            │  │
│  │               目标: 越小越好                              │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  f_resource   = |avg_busy - target_util| / target_util  │  │
│  │               target_util = 0.7                         │  │
│  │               偏离最优利用率越远, 惩罚越大                │  │
│  │               目标: 越小越好                              │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                              │
│  权重:                                                       │
│  w₁ = 0.40  (延迟, 3个参数都直接影响延迟, 权重最大)           │
│  w₂ = 0.30  (吞吐量)                                        │
│  w₃ = 0.15  (checkpoint 开销)                                │
│  w₄ = 0.15  (资源利用率)                                     │
│                                                              │
│  J ∈ [0, +∞), 越小越好                                      │
│  理想值 J ≈ 0 (延迟低、吞吐高、CP 快、资源适中)              │
└──────────────────────────────────────────────────────────────┘

```

### 4.2约束条件

```text
┌──────────────────────────────────────────────────────────────┐
│                   硬约束 (违反则丢弃配置)                       │
│                                                              │
│  C1: latency_p99  ≤ 5000 ms     (SLA 延迟约束)              │
│  C2: backpressure ≤ 0.80        (反压不能过高)               │
│                                                              │
│  软约束 (通过目标函数惩罚)                                     │
│  S1: checkpoint_duration 过长 → f_checkpoint 自然增大        │
│  S2: busy_ratio 偏离 0.7 → f_resource 自然增大               │
└──────────────────────────────────────────────────────────────┘
```

## 五、代理模型设计
### 5.1 两阶段模型策略
```text
┌──────────────────────────────────────────────────────────────────┐
│                    两阶段代理模型                                  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  Phase 1: 冷启动探索期                                      │  │
│  │  触发条件: 有效观测数 n < N_warm (N_warm = 10)               │  │
│  │                                                            │  │
│  │  采样策略: 拉丁超立方采样 (LHS)                               │  │
│  │  ┌──────────────────────────────────────────────────┐      │  │
│  │  │  在 3 维参数空间中生成 10 个均匀分布的采样点         │      │  │
│  │  │  确保每个参数维度都被充分探索                        │      │  │
│  │  │  第 1 个点为默认配置 (基线)                          │      │  │
│  │  │  第 2~10 个点为 LHS 生成的探索点                    │      │  │
│  │  └──────────────────────────────────────────────────┘      │  │
│  │                                                            │  │
│  │  代理模型: Random Forest                                    │  │
│  │  ┌──────────────────────────────────────────────────┐      │  │
│  │  │  RandomForestRegressor(                           │      │  │
│  │  │    n_estimators=100,                              │      │  │
│  │  │    max_depth=10,                                  │      │  │
│  │  │    min_samples_leaf=2,                            │      │  │
│  │  │    random_state=42                                │      │  │
│  │  │  )                                                │      │  │
│  │  │                                                    │      │  │
│  │  │  不确定性估计:                                      │      │  │
│  │  │    σ(z) = std(各棵树的预测值)                       │      │  │
│  │  │    → 用于 Phase 1 后期的 EI 过渡                    │      │  │
│  │  └──────────────────────────────────────────────────┘      │  │
│  │                                                            │  │
│  │  Phase 1 结束条件: 积累 10 个有效观测                        │  │
│  └────────────────────────────────────────────────────────────┘  │
│                              │                                    │
│                              ▼                                    │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  Phase 2: 贝叶斯优化期                                      │  │
│  │  触发条件: 有效观测数 n ≥ 10                                 │  │
│  │                                                            │  │
│  │  代理模型: Gaussian Process                                  │  │
│  │  ┌──────────────────────────────────────────────────┐      │  │
│  │  │  GaussianProcessRegressor(                        │      │  │
│  │  │    kernel = C(1.0) * RBF(                         │      │  │
│  │  │      length_scale=[1.0]*19,                       │      │  │
│  │  │      length_scale_bounds=(1e-3, 1e3)              │      │  │
│  │  │    ) + WhiteKernel(                               │      │  │
│  │  │      noise_level=0.1,                             │      │  │
│  │  │      noise_level_bounds=(1e-5, 1e1)               │      │  │
│  │  │    ),                                              │      │  │
│  │  │    n_restarts_optimizer=5,                        │      │  │
│  │  │    normalize_y=True,                              │      │  │
│  │  │    alpha=1e-6                                     │      │  │
│  │  │  )                                                │      │  │
│  │  │                                                    │      │  │
│  │  │  输入: z ∈ R^19 = [X_norm(16) + θ_norm(3)]        │      │  │
│  │  │  输出: μ(z) 预测均值, σ(z) 预测标准差              │      │  │
│  │  └──────────────────────────────────────────────────┘      │  │
│  │                                                            │  │
│  │  每轮流程:                                                  │  │
│  │  1. 全部历史有效观测 → 重新拟合 GP                           │  │
│  │  2. 生成 200 个候选参数 θ_cand                              │  │
│  │  3. 拼接当前 X_norm → z_cand                               │  │
│  │  4. GP 预测 μ, σ → 计算 EI → 硬约束过滤                    │  │
│  │  5. 安全检查 → 效率阈值 → 选定 θ*                           │  │
│  └────────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘

```

### 5.2 采集函数 EI

```text
┌──────────────────────────────────────────────────────────────────┐
│                  Expected Improvement (EI)                        │
│                                                                  │
│  解析解:                                                          │
│                                                                  │
│         ⎧ (J_best - μ - ξ) · Φ(Z) + σ · φ(Z)   if σ > 0       │
│  EI(θ) =⎨                                                       │
│         ⎩ 0                                       if σ = 0       │
│                                                                  │
│    Z = (J_best - μ(θ) - ξ) / σ(θ)                               │
│    μ(θ)  = GP 预测均值                                            │
│    σ(θ)  = GP 预测标准差                                          │
│    Φ(·)  = 标准正态 CDF                                          │
│    φ(·)  = 标准正态 PDF                                          │
│    J_best = 历史所有有效观测中的最小目标值                           │
│                                                                  │
│  ξ 衰减策略 (探索→利用):                                          │
│    ξ(n) = max(0.001, 0.05 × exp(-0.05 × (n - N_warm)))          │
│    n = 当前有效观测数, N_warm = 10                                │
│    冷启动期 ξ ≈ 0.05 (鼓励探索)                                   │
│    后期     ξ → 0.001 (加速收敛)                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 5.3 候选参数生成与筛选
```text
┌──────────────────────────────────────────────────────────────────┐
│                  候选参数生成与筛选流程                              │
│                                                                  │
│  Step 1: 生成候选集 (共 200 个)                                   │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │  来源A: 随机均匀采样 (140 个, 70%)                        │    │
│  │    在 3 维归一化空间 [0,1]^3 中均匀随机生成                │    │
│  │    → 全局探索, 防止陷入局部最优                            │    │
│  │                                                          │    │
│  │  来源B: 当前最优邻域采样 (40 个, 20%)                      │    │
│  │    以 θ_best 为中心, 在 ±15% 范围内高斯扰动               │    │
│  │    → 局部精细搜索, 加速收敛                                │    │
│  │                                                          │    │
│  │  来源C: 历史 Top-5 邻域采样 (20 个, 10%)                  │    │
│  │    以历史前 5 优配置为中心, 各生成 4 个邻域点              │    │
│  │    → 利用历史经验, 避免遗忘好配置                          │    │
│  └──────────────────────────────────────────────────────────┘    │
│                                                                  │
│  Step 2: 反归一化 + 离散化                                        │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │  θ_real = θ_norm × (max - min) + min                     │    │
│  │                                                          │    │
│  │  离散化:                                                  │    │
│  │    checkpoint_interval = round(val / 5000) × 5000         │    │
│  │    buffer_timeout      = round(val / 10) × 10             │    │
│  │    watermark_delay     = round(val / 1000) × 1000         │    │
│  │                                                          │    │
│  │  边界裁剪:                                                │    │
│  │    checkpoint_interval = clamp(val, 5000, 120000)         │    │
│  │    buffer_timeout      = clamp(val, 1, 200)               │    │
│  │    watermark_delay     = clamp(val, 1000, 30000)          │    │
│  └──────────────────────────────────────────────────────────┘    │
│                                                                  │
│  Step 3: GP 预测 + 硬约束预过滤                                   │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │  对每个候选 θ_cand:                                       │    │
│  │    z_cand = [X_current_norm, θ_cand_norm]                 │    │
│  │    μ, σ = GP.predict(z_cand)                              │    │
│  │                                                          │    │
│  │  过滤规则:                                                │    │
│  │  ① 乐观估计下界: μ - 2σ > J_best × 1.5 → 丢弃           │    │
│  │     (即使最乐观估计也比当前最优差 50%)                      │    │
│  │  ② 去重: 离散化后与已测试配置完全相同 → 丢弃              │    │
│  └──────────────────────────────────────────────────────────┘    │
│                                                                  │
│  Step 4: 计算 EI + 安全选择                                      │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │  对过滤后的候选集:                                         │    │
│  │    EI(θ_cand) = compute_ei(μ, σ, J_best, ξ)              │    │
│  │                                                          │    │
│  │  按 EI 降序排列                                           │    │
│  │                                                          │    │
│  │  安全选择策略 (借鉴 Demeter SB 思想):                     │    │
│  │    取 Top-3 中 σ 最小的那个 → θ*                          │    │
│  │    → 在高 EI 候选中选最稳妥的                              │    │
│  │    → 平衡收益与风险                                       │    │
│  └──────────────────────────────────────────────────────────┘    │
│                                                                  │
│  Step 5: 效率阈值检查 (ET)                                       │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │  ET = 0.08 (8%)                                          │    │
│  │                                                          │    │
│  │  predicted_improvement = (J_current - μ(θ*)) / J_current │    │
│  │                                                          │    │
│  │  if predicted_improvement < ET:                           │    │
│  │    → 不执行变更, 保持当前配置                              │    │
│  │    → 返回 None, 主循环跳过执行阶段                         │    │
│  │                                                          │    │
│  │  if predicted_improvement >= ET:                           │    │
│  │    → 返回 {θ_next: θ*, ei: EI(θ*), mu: μ, sigma: σ}     │    │
│  └──────────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────────┘
```

## 七、 安全机制
### 7.1 三层安全屏障
```text
┌──────────────────────────────────────────────────────────────────┐
│                      三层安全屏障                                  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  第一层: 参数边界硬约束                                      │  │
│  │                                                            │  │
│  │  所有参数始终 clamp 在 [min, max] 范围内                     │  │
│  │  离散化步长保证参数值合法                                    │  │
│  │  任何来源的参数建议都必须经过边界检查                          │  │
│  │                                                            │  │
│  │  checkpoint_interval = clamp(round(val/5000)*5000,         │  │
│  │                              5000, 120000)                 │  │
│  │  buffer_timeout      = clamp(round(val/10)*10, 1, 200)     │  │
│  │  watermark_delay     = clamp(round(val/1000)*1000,         │  │
│  │                              1000, 30000)                  │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  第二层: GP 置信度检查                                       │  │
│  │                                                            │  │
│  │  σ_threshold = 2.0 × std(所有历史有效 J 值)                 │  │
│  │                                                            │  │
│  │  if σ(θ*) > σ_threshold:                                   │  │
│  │    → 不确定性过高, 说明参数空间未被充分探索                   │  │
│  │    → 降级为保守模式:                                        │  │
│  │      θ_cand = θ_best ± uniform(-0.10, +0.10) × range      │  │
│  │      只在已知安全区域 ±10% 范围内微调                        │  │
│  │    → 避免跳入完全未知的参数区域                               │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  第三层: 运行时回滚机制                                       │  │
│  │                                                            │  │
│  │  应用新参数后, 在前 60s 进行快速健康检查:                     │  │
│  │  每 15s 检查一次, 共 4 次                                    │  │
│  │                                                            │  │
│  │  ROLLBACK 触发条件 (连续 2 次不通过才触发):                   │  │
│  │  ① latency_p99 > 2 × SLA = 10000ms                        │  │
│  │  ② throughput < 0.3 × baseline_throughput                   │  │
│  │  ③ backpressure_ratio > 0.9                                 │  │
│  │                                                            │  │
│  │  回滚执行:                                                  │  │
│  │    1. Savepoint → Cancel → 用 θ_best 重新提交               │  │
│  │    2. 标记本轮观测 is_valid = False                          │  │
│  │    3. rollback_count += 1                                   │  │
│  │                                                            │  │
│  │  连续回滚保护:                                               │  │
│  │    if rollback_count >= 3:                                  │  │
│  │      → 暂停 ML Tuner 30 分钟                                │  │
│  │      → 使用 θ_best 稳定运行                                  │  │
│  │      → 30 分钟后重置计数器, 恢复调优                          │  │
│  └────────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘

```

## 十二、主控循环伪代码
```text
┌──────────────────────────────────────────────────────────────────┐
│                ml_tuner_controller.py 主循环                      │
│                                                                  │
│  def main():                                                     │
│    # ═══════════════════════════════                             │
│    # 初始化所有组件                                               │
│    # ═══════════════════════════════                             │
│    config    = load_config("ml_tuner_config.yaml")               │
│    space     = ParameterSpace(config.parameter_space)  # 3维     │
│    store     = ObservationStore(config.data)                     │
│    collector = ObservationCollector(config.flink)                │
│    engineer  = FeatureEngineer(config)                           │
│    model     = SurrogateModel(config.surrogate)                  │
│    optimizer = BayesianOptimizer(config.optimizer, space)        │
│    safety    = SafetyChecker(config.safety, config.constraints)  │
│    executor  = ParamExecutor(config.flink, config.coordination)  │
│    coord     = Coordinator(config.coordination)                  │
│                                                                  │
│    # ═══════════════════════════════                             │
│    # 恢复历史状态                                                 │
│    # ═══════════════════════════════                             │
│    store.load()                                                  │
│    state = load_state(config.data.tuner_state_json)              │
│    θ_current = state.get("θ_current", space.get_defaults())     │
│    round_id  = state.get("round_id", 0)                         │
│    N_WARM    = config.surrogate.warm_up_observations  # 10      │
│                                                                  │
│    # ═══════════════════════════════                             │
│    # 冷启动 LHS 采样计划                                         │
│    # ═══════════════════════════════                             │
│    n_valid = store.count_valid()                                 │
│    if n_valid < N_WARM:                                          │
│      lhs_plan = space.latin_hypercube(N_WARM)                   │
│      lhs_plan[0] = space.get_defaults()  # 第0组是基线           │
│                                                                  │
│    # ═══════════════════════════════                             │
│    # 首次启动等待系统稳定                                         │
│    # ═══════════════════════════════                             │
│    if round_id == 0:                                             │
│      log("首次启动, 等待 5min 让系统稳定")                         │
│      sleep(300)                                                  │
│                                                                  │
│    # ══════════════════════════════════════════════              │
│    #                   主循环                                    │
│    # ══════════════════════════════════════════════              │
│    while True:                                                   │
│      round_id += 1                                               │
│      n_valid = store.count_valid()                               │
│      phase = "explore" if n_valid < N_WARM else "optimize"      │
│      log(f"===== Round {round_id} | {phase} | "                 │
│          f"valid={n_valid} =====")                               │
│                                                                  │
│      # ─────────────────────────────                             │
│      # Step 1: 采集当前环境特征                                   │
│      # ─────────────────────────────                             │
│      try:                                                        │
│        raw_samples = collector.collect_window(                   │
│          window_sec  = config.timing.observation_window_seconds, │
│          interval_sec = config.timing.sample_interval_seconds   │
│        )  # 5min, 每30s采一次, 共10个样本                        │
│                                                                  │
│        X_raw  = engineer.aggregate_median(raw_samples)  # 16维   │
│        X_norm = engineer.normalize_features(X_raw)      # 16维   │
│                                                                  │
│      except Exception as e:                                      │
│        log(f"指标采集失败: {e}, 跳过本轮")                          │
│        sleep(60)                                                 │
│        continue                                                  │
│                                                                  │
│      # ─────────────────────────────                             │
│      # Step 2: 计算当前配置的目标值                                │
│      # ─────────────────────────────                             │
│      J_current, J_parts = engineer.compute_objective(X_raw)      │
│      # J_parts = {J_latency, J_throughput, J_checkpoint,         │
│      #            J_resource}                                    │
│                                                                  │
│      sla_violated = X_raw["latency_p99"] > \                     │
│                     config.constraints.max_latency_p99_ms        │
│      bp_violated  = X_raw["backpressure_ratio"] > \              │
│                     config.constraints.max_backpressure           │
│      is_valid = not (sla_violated or bp_violated)                │
│                                                                  │
│      # ─────────────────────────────                             │
│      # Step 3: 记录当前观测                                      │
│      # ─────────────────────────────                             │
│      obs = {                                                     │
│        "round_id": round_id,                                     │
│        "timestamp": now_iso(),                                   │
│        "phase": phase,                                           │
│        **X_raw,                                                  │
│        **θ_current,                                              │
│        "J_total": J_current,                                     │
│        **J_parts,                                                │
│        "sla_violated": sla_violated,                             │
│        "bp_violated": bp_violated,                               │
│        "is_valid": is_valid,                                     │
│      }                                                           │
│      store.add(obs)                                              │
│      store.save()                                                │
│                                                                  │
│      log(f"J={J_current:.4f} | J_best={store.J_best:.4f} | "    │
│          f"valid={is_valid}")                                    │
│      log(f"  lat={J_parts['J_latency']:.3f} "                   │
│          f"thr={J_parts['J_throughput']:.3f} "                   │
│          f"ckp={J_parts['J_checkpoint']:.3f} "                   │
│          f"res={J_parts['J_resource']:.3f}")                     │
│                                                                  │
│      # ─────────────────────────────                             │
│      # Step 4: 选择下一组参数                                     │
│      # ─────────────────────────────                             │
│                                                                  │
│      if phase == "explore":                                      │
│        # ── Phase 1: LHS 探索 ──                                 │
│        θ_next = lhs_plan[n_valid]                                │
│        log(f"LHS 探索 [{n_valid+1}/{N_WARM}]: θ={θ_next}")       │
│                                                                  │
│      else:                                                       │
│        # ── Phase 2: 贝叶斯优化 ──                                │
│                                                                  │
│        # 4a. 准备训练数据                                         │
│        Z_train, J_train = store.get_training_data(engineer)      │
│        # Z_train: (n, 19),  J_train: (n,)                       │
│                                                                  │
│        # 4b. 拟合代理模型                                         │
│        try:                                                      │
│          model.fit(Z_train, J_train, model_type="gp")            │
│          log(f"GP 拟合成功, 训练样本={len(J_train)}")               │
│        except Exception as e:                                    │
│          log(f"GP 拟合失败({e}), 降级 RF")                         │
│          model.fit(Z_train, J_train, model_type="rf")            │
│                                                                  │
│        # 4c. 贝叶斯优化建议                                       │
│        suggestion = optimizer.suggest(                            │
│          model  = model,                                         │
│          X_norm = X_norm,                                        │
│          J_best = store.J_best,                                  │
│          n_obs  = n_valid                                        │
│        )                                                         │
│        # suggestion = {θ_next, ei, mu, sigma} or None            │
│                                                                  │
│        # 4d. 效率阈值检查                                         │
│        if suggestion is None:                                    │
│          log(f"EI 改善 < ET({config.optimizer.ET}), skip")        │
│          save_state(round_id, θ_current)                         │
│          continue                                                │
│                                                                  │
│        θ_next = suggestion["θ_next"]                             │
│                                                                  │
│        # 4e. GP 置信度检查                                        │
│        if not safety.check_confidence(                            │
│          sigma=suggestion["sigma"], store=store                   │
│        ):                                                        │
│          log("置信度不足, 降级保守搜索")                             │
│          θ_next = optimizer.suggest_conservative(                 │
│            θ_best = store.θ_best,                                │
│            range_pct = config.safety.conservative_range          │
│          )                                                       │
│                                                                  │
│        log(f"BO 建议: θ={θ_next} | "                              │
│            f"EI={suggestion['ei']:.4f}")                         │
│                                                                  │
│      # ─────────────────────────────                             │
│      # Step 5: 参数相同则跳过                                     │
│      # ─────────────────────────────                             │
│      if θ_next == θ_current:                                     │
│        log("新参数与当前相同, 跳过执行")                            │
│        save_state(round_id, θ_current)                           │
│        continue                                                  │
│                                                                  │
│      # ─────────────────────────────                             │
│      # Step 6: 执行参数变更                                      │
│      # ─────────────────────────────                             │
│      try:                                                        │
│        # 6a. 获取协调锁, 暂停 DS2                                 │
│        log("尝试获取协调锁...")                                     │
│        acquired = coord.acquire_lock(                             │
│          timeout=config.coordination.lock_timeout_seconds        │
│        )                                                         │
│        if not acquired:                                          │
│          log("获取锁超时, 放弃本轮")                                │
│          save_state(round_id, θ_current)                         │
│          continue                                                │
│                                                                  │
│        coord.pause_ds2()                                         │
│        log("已获取锁, DS2 已暂停")                                 │
│                                                                  │
│        # 6b. 获取当前并行度 (保持 DS2 已调好的值)                   │
│        current_parallelism = executor.get_current_parallelism()   │
│        log(f"当前并行度: {current_parallelism}")                    │
│                                                                  │
│        # 6c. Savepoint → Cancel → Resubmit                      │
│        executor.apply_params(                                    │
│          θ_new = θ_next,                                         │
│          parallelism = current_parallelism                       │
│        )                                                         │
│        log(f"作业已用新参数重启: {θ_next}")                         │
│                                                                  │
│        # 6d. Warm-up                                             │
│        warm_up = config.timing.warm_up_seconds  # 90s            │
│        log(f"Warm-up {warm_up}s ...")                             │
│        sleep(warm_up)                                            │
│                                                                  │
│        # 6e. 健康检查                                             │
│        log("开始健康检查...")                                       │
│        healthy = safety.health_check(                             │
│          collector = collector,                                   │
│          interval  = config.safety.health_check.interval_seconds,│
│          count     = config.safety.health_check.count,           │
│          sla_limit = config.constraints.max_latency_p99_ms \     │
│                      * config.safety.rollback.latency_multiplier,│
│          thr_floor = store.get_baseline_throughput() \            │
│                      * config.safety.rollback.throughput_drop,   │
│          bp_limit  = config.safety.rollback.backpressure_limit,  │
│          consec    = config.safety.rollback.consecutive_failures │
│        )                                                         │
│                                                                  │
│        if not healthy:                                            │
│          # ── 回滚 ──                                             │
│          log("!! 健康检查失败, 触发回滚 !!")                        │
│          executor.apply_params(                                   │
│            θ_new = store.θ_best,                                 │
│            parallelism = current_parallelism                     │
│          )                                                       │
│          θ_current = store.θ_best                                │
│          safety.record_rollback()                                │
│          log(f"已回滚到 θ_best={store.θ_best}, "                  │
│              f"rollback_count={safety.rollback_count}")           │
│                                                                  │
│          # 标记本轮观测为无效                                      │
│          store.mark_last_invalid()                               │
│          store.save()                                            │
│                                                                  │
│          # 连续回滚保护                                            │
│          if safety.should_pause():                                │
│            pause_min = config.safety.rollback.pause_duration_min │
│            log(f"连续回滚 {safety.rollback_count} 次, "           │
│                f"暂停 {pause_min}min")                            │
│            coord.resume_ds2()                                    │
│            coord.release_lock()                                  │
│            sleep(pause_min * 60)                                 │
│            safety.reset_rollback_count()                         │
│            log("暂停结束, 恢复调优")                                │
│            save_state(round_id, θ_current)                       │
│            continue                                              │
│                                                                  │
│        else:                                                     │
│          # ── 健康检查通过 ──                                      │
│          θ_current = θ_next                                      │
│          safety.reset_rollback_count()                            │
│          log(f"健康检查通过, θ_current={θ_current}")               │
│                                                                  │
│      except Exception as e:                                      │
│        # ── 执行异常兜底 ──                                        │
│        log(f"执行异常: {e}")                                       │
│        try:                                                      │
│          log("尝试回退到 θ_best...")                                │
│          executor.apply_params(                                   │
│            θ_new = store.θ_best,                                 │
│            parallelism = current_parallelism                     │
│          )                                                       │
│          θ_current = store.θ_best                                │
│        except Exception as e2:                                   │
│          log(f"回退也失败: {e2}, 等待人工干预")                      │
│          coord.resume_ds2()                                      │
│          coord.release_lock()                                    │
│          sleep(600)  # 等10min再重试                              │
│          save_state(round_id, θ_current)                         │
│          continue                                                │
│                                                                  │
│      finally:                                                    │
│        # ── 释放锁, 恢复 DS2 ──                                   │
│        coord.resume_ds2()                                        │
│        coord.release_lock()                                      │
│        log("锁已释放, DS2 已恢复")                                 │
│                                                                  │
│      # ─────────────────────────────                             │
│      # Step 7: 持久化状态                                         │
│      # ─────────────────────────────                             │
│      save_state(round_id, θ_current)                             │
│      save_current_params(θ_current,                              │
│                          config.data.tuner_params_yaml)          │
│      log(f"状态已保存, 进入下一轮")                                 │
│                                                                  │
│      # ── 回到 Step 1, 进入下一个观测窗口 ──                      │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘


```
